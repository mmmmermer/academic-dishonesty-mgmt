# 服务器上线部署指南（供教师办公室使用）

面向「在学校服务器上部署，供办公室教师通过浏览器访问」的改造方向与操作步骤。不改变业务代码，只涉及环境、运行方式、网络与运维。

---

## 一、要改动的方向概览

| 方向 | 说明 |
|------|------|
| **1. 服务器环境** | 安装 Python 3.9+、依赖，用虚拟环境隔离 |
| **2. 项目与数据目录** | 确定项目路径，保证 database.db、logs、backups 可写、权限合适 |
| **3. 首次初始化** | 执行 init_db.py，**务必修改默认管理员密码** |
| **4. 监听与端口** | 让 Streamlit 监听 `0.0.0.0`，办公室内网可访问；固定端口（如 8501） |
| **5. 进程常驻** | 用 systemd（Linux）或计划任务/服务（Windows）保证崩溃或重启后自动拉起 |
| **6. 防火墙** | 放行 Streamlit 端口（或反向代理端口） |
| **7. 备份与日常运维** | 定期备份 database.db 和 backups/，会看 logs/app.log |
| **8. 可选：反向代理** | 用 Nginx 挂到 80 端口、统一入口，便于以后加 HTTPS |

下面按顺序说明。

## 二、服务器环境

### 2.1 系统与 Python

- **系统**：需求文档写的是「学校办公室局域网 (Windows/Linux Server)」，两种都支持；Linux 更便于用 systemd 做服务。
- **Python**：3.9 或以上，建议 3.10/3.11。
  - Linux 示例：`sudo apt install python3.10 python3.10-venv`（Debian/Ubuntu）或 `yum install python3.11`（CentOS 等）。
  - Windows：从 python.org 安装，勾选「Add to PATH」。

### 2.2 项目目录与虚拟环境

- 把整份项目放到服务器固定目录，例如：
  - Linux：`/opt/hustsystem` 或 `/home/deploy/hustsystem`
  - Windows：`C:\Apps\hustsystem`
- 在项目根目录建虚拟环境并装依赖：

```bash
cd /opt/hustsystem   # 或你的实际路径
python3 -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

- 确认能正常启动（先不对外）：

```bash
python -m streamlit run app.py
```

Ctrl+C 停掉后再做下面的「监听与端口」配置。

---

## 三、项目与数据目录

- **database.db**、**backups/**、**logs/** 都在项目根目录下生成（当前代码已如此），无需改代码。
- 需要保证：
  - 运行 Streamlit 的用户对该目录有**读写的权限**（创建/写 database.db、backups、logs）。
  - 若用 systemd/服务跑，运行用户一般是 `root` 或专用用户（如 `www-data`、`deploy`），对应把项目目录属主或权限设好。
- **logs/** 已在 .gitignore 中，部署后会自动创建；若需日志轮转，可后续用 logrotate（Linux）或计划任务压缩旧日志。

## 四、首次初始化与安全

### 4.1 建表与默认管理员

在项目根、激活虚拟环境后执行：

```bash
python init_db.py
```

- 会创建所有表，并写入默认管理员：**用户名 admin，密码 123456**。

### 4.2 务必修改默认密码

- 上线后**第一时间**用浏览器登录 admin，在「用户管理」里给 admin 做一次**密码重置**，或新建管理员后停用 admin。
- 避免默认 123456 被内网其他人试出。

## 五、监听地址与端口（办公室能访问）

- 默认 `streamlit run app.py` 只监听本机，办公室其他电脑访问不到。
- 需要绑定到**所有网卡**并固定端口，例如：

```bash
python -m streamlit run app.py --server.address=0.0.0.0 --server.port=8501
```

- **参数说明**：
  - `--server.address=0.0.0.0`：允许通过服务器内网 IP（如 192.168.1.100）访问。
  - `--server.port=8501`：固定端口，便于防火墙和反向代理配置。

- 教师访问地址示例：`http://192.168.1.100:8501`（把 IP 换成实际服务器内网 IP）。

- **可选**：在项目根建一个启动脚本，方便以后用，例如 `run.sh`（Linux）或 `run.bat`（Windows），内容就是上面这一行（前面加激活 venv 的命令）。

## 六、进程常驻（崩溃或重启后自动拉起）

### 6.1 Linux：systemd

- 新建服务文件，例如 `/etc/systemd/system/hustsystem.service`：

```ini
[Unit]
Description=学术失信人员管理系统 Streamlit
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/hustsystem
Environment="PATH=/opt/hustsystem/venv/bin"
ExecStart=/opt/hustsystem/venv/bin/streamlit run app.py --server.address=0.0.0.0 --server.port=8501
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

- 把 `WorkingDirectory`、`Environment`、`ExecStart` 里的路径改成你的项目路径。
- 然后执行：

```bash
sudo systemctl daemon-reload
sudo systemctl enable hustsystem
sudo systemctl start hustsystem
sudo systemctl status hustsystem
```

- 以后重启服务器，服务会自动起来；崩溃也会被 `Restart=always` 拉起来。

### 6.2 Windows

- 可用**计划任务**：触发器选「计算机启动时」，操作里运行 `run.bat`（内容为激活 venv + 上面那行 streamlit 命令）。
- 或使用 NSSM 等工具把 `streamlit run ...` 注册为 Windows 服务，便于「启动类型：自动」和崩溃重启。

## 七、防火墙

- 办公室电脑要能访问服务器上的 Streamlit，需要在服务器（或学校出口防火墙）上**放行对应端口**。
- 若直接访问 Streamlit：放行 **8501**（或你用的端口）。
- 若用 Nginx 反向代理（见第八节）：放行 **80**（以及将来 HTTPS 的 443）。
- Linux 示例（firewalld）：`sudo firewall-cmd --permanent --add-port=8501/tcp && sudo firewall-cmd --reload`。

---

## 八、备份与日常运维

### 8.1 备份

- **database.db**：系统已有「系统维护 → 数据库备份下载」，管理员可手动下载；同时建议在服务器上做**定期自动备份**。
- **backups/**：启动时若执行了 `auto_backup()`，会在项目下生成 `backups/database_时间戳.db`，可再配合 cron/计划任务把 `backups/` 拷到别的盘或另一台机器。
- 示例 cron（每天凌晨 2 点复制一份到 /backup 目录）：

```bash
0 2 * * * cp -r /opt/hustsystem/backups /backup/hustsystem-$(date +\%Y\%m\%d)
```

### 8.2 日常运维

- **看日志**：异常和部分操作会写在 `logs/app.log`，排查问题先看这里。
- **重启服务**：Linux 用 `sudo systemctl restart hustsystem`；Windows 重启计划任务或对应服务。
- **更新代码**：拉新代码后重启 Streamlit 进程即可；若表结构有变，需按项目说明执行迁移或脚本。

## 九、可选：Nginx 反向代理（80 端口、以后可加 HTTPS）

- 教师记不住端口时，可用 Nginx 把 80 转到 Streamlit，访问 `http://服务器IP` 即可。
- 示例配置（`/etc/nginx/sites-available/hustsystem`，再软链到 sites-enabled）：

```nginx
server {
    listen 80;
    server_name _;  # 或填服务器内网 IP/域名
    location / {
        proxy_pass http://127.0.0.1:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

- 重载 Nginx 后，教师访问 `http://服务器IP` 即等价于访问 8501。
- 若将来要做 HTTPS，可在同一 server 里加 `listen 443 ssl` 和证书配置，并保留上述 proxy 配置。
