# 华中科技大学学术道德监督管理系统 — 用户使用说明

本文档面向最终用户，介绍系统功能、使用方式、功能边界及技术概况。

---

## 一、产品概述

### 1.1 系统简介

**华中科技大学学术道德监督管理系统**（Academic Dishonesty Management System）是一套面向高校的 Web 应用，用于：

- **名单管理**：维护校内学术失信/违规人员名单（生效中、已撤销）。
- **查询与比对**：教师按学号单条查询或上传 Excel 批量比对，判断学生是否在生效名单中。
- **审计与备份**：记录操作日志，支持数据备份与恢复（视部署方式而定）。

### 1.2 适用对象与场景

| 对象     | 典型场景 |
|----------|----------|
| 教务处/学工 | 导入名单、维护生效/撤销状态、查看统计、用户与系统维护 |
| 教师/辅导员 | 单条查询学生是否在名单中、批量比对名单（如答辩/评奖前） |
| 系统管理员 | 用户账号、密码、备份与恢复、审计日志查看与导出 |

### 1.3 访问方式

- **系统地址**：**http://8.148.72.200:8501**（部署后的访问网址，由部署单位提供；若改用域名或端口，以实际为准）。
- **浏览器**：建议使用 Chrome、Edge、Safari 等现代浏览器。
- **网络**：需能访问部署该系统的服务器（校园网或公网视部署而定）。

## 二、角色与权限

系统有两种角色，权限边界清晰：

| 角色     | 权限概述 |
|----------|----------|
| **教师** | 仅可查询：单条查询、批量比对、查看个人操作记录；不可增删改名单与用户。 |
| **管理员** | 名单的增删改、导入、生效/撤销/恢复；用户管理；仪表盘；审计日志；备份与恢复（视部署）。 |

同一账号仅对应一种角色；登录后页面与侧栏功能按角色展示。

## 三、登录与安全

### 3.1 登录

- **工号**：登录账号（由管理员创建或使用默认管理员账号）。
- **密码**：首次使用请使用管理员告知的初始密码，并尽快修改（管理员账号请在首次登录后修改默认密码）。

### 3.2 安全机制

- **会话超时**：长时间无操作（默认约 30 分钟）将自动退出，需重新登录。
- **登录保护**：同一账号连续多次登录失败后，将进入冷却期（如 5 分钟内不可再次尝试）。
- **水印**：页面带有当前登录人姓名与工号的水印，用于防截屏与责任追溯。
- **审计**：关键操作（登录、查询、导入、增删改、备份恢复等）均会记录到审计日志。

### 3.3 退出登录

点击侧栏底部 **「退出登录」** 即可安全退出。

## 四、教师端功能说明

教师登录后进入 **「学术诚信档案查询」**，仅可查看**生效中**的失信记录。

### 4.1 单条查询

- **入口**：侧栏选择「单条查询」。
- **操作**：输入学生**学号**、**姓名**、**姓名拼音全拼**或**姓名首字母**后查询。
- **多人查询**：可使用换行、逗号、顿号、分号分隔多个人；如需联合校验，可写成「姓名 + 学号」。
- **姓名空格兼容**：输入中出现空格时，系统会自动按同一姓名规范化处理，例如「张 伟」可命中「张伟」。
- **结果**：
  - 若在生效名单中：展示该生的姓名、学号、专业、原因、处分日期等。
  - 若未在名单中：提示「未查询到失信记录，所查学生信用良好」。
  - 若是拼音、首字母或简称命中：系统会提示该结果包含模糊匹配，请结合学号进一步核实。

### 4.2 批量比对

- **入口**：侧栏选择「批量比对」。
- **操作**：上传 Excel 文件（.xlsx 或 .xls），表格中**至少包含一列「学号」**；系统自动与生效名单比对。
- **结果**：
  - **有命中**：红色提示命中条数，表格展示命中记录（分页），可下载比对结果 Excel。
  - **无命中**：绿色提示名单内学生信用良好，可查看/下载未命中名单。
- **说明**：学号会自动清洗（去空格、全角转半角等）；单次上传行数、文件大小有限制（见第六节）。

### 4.3 个人记录

- **入口**：侧栏选择「个人记录」。
- **内容**：当前登录教师**最近的操作历史**（如单条查询、批量比对），最多展示 100 条；包含时间、类型、对象、简要详情。

## 五、管理员端功能说明

管理员登录后，侧栏有 **「功能板块」** 选择：仪表盘、名单管理、用户管理、系统维护。

### 5.1 仪表盘

- **名单概览**：名单总数、生效中数量、已撤销数量。
- **图表**：按专业分布（饼图）、按处分年份分布（柱状图）；近期名单变动列表。
- **说明**：数据有短时缓存，恢复数据后可能需稍等或刷新后更新。

### 5.2 名单管理

- **批量导入**  
  - Excel 需包含列：**姓名、学号、专业、原因、处分时间**（列名需一致）。  
  - 学号唯一：已存在则更新，不存在则新增；学号为空的行会跳过并可在「上次导入结果」中查看。  
  - 支持 .xlsx、.xls；单次导入行数、文件大小有限制（见第六节）。

- **手动新增**  
  - 单条录入：姓名、学号、专业、原因、处分日期；学号不可与已有记录重复。

- **生效名单**  
  - 支持按姓名、姓名拼音、姓名首字母、学号、专业筛选（留空表示不限制）；分页、排序、每页条数可调。  
  - 姓名多项筛选建议使用换行、逗号、顿号或分号分隔，避免把姓名内部空格误当作多个关键词。  
  - 可对指定学号执行：软删除（设为已撤销）、编辑。

- **已撤销名单**  
  - 同样支持筛选、分页、排序。  
  - 可对指定学号执行：恢复为生效。

- **按学号操作（含违规通报 PDF 上传）**  
  - 删除（软删除）、编辑、恢复为生效：在对应展开区输入学号后操作。
  - **PDF 附件管理**：在单条编辑时可上传该对象的处分决定 PDF。系统已内置“垃圾回收”机制，多次修改或替换 PDF 时，旧的 PDF 文件会被自动清理释放，请放心使用。
  - **初始化名单**：将所有生效记录设为已撤销，生效名单清空；需二次确认，请谨慎使用。

- **导出**  
  - 生效名单、已撤销名单均可按当前筛选与排序导出 Excel（有最大条数限制，见第六节）。

### 5.3 用户管理

- **用户列表**：查看所有用户（工号、姓名、角色、启用状态）。
- **新增用户**：填写工号（唯一）、姓名、密码、角色（管理员/教师）；工号不可重复。
- **密码重置**：选择用户后重置密码，新密码由管理员告知用户。
- **启用/禁用**：禁用后该账号无法登录；**不能禁用当前登录账号**。

### 5.4 系统维护

- **审计日志**  
  - 可按操作人、操作类型、日期筛选；表格展示最近若干条（如 500 条）。
  - **一键导出机制**：导出海量日志时采用了分层保护机制，需先点击“准备导出数据”，待系统将全表数据提取并缓存后，再点击“下载 Excel”，避免重复查询导致卡顿。

- **数据库备份与恢复**（视部署方式）  
  - **自动备份告警**：系统每天会自动备份一次数据。若因服务器磁盘满或权限问题导致备份失败，此页面顶部会出现显眼的红色高亮告警，提示管理员及时检修。
  - **SQLite 部署**：可下载当前数据库 .db 文件作为手动备份；可上传备份的 .db 文件执行恢复（会覆盖当前数据，需勾选确认）。  
  - **MySQL/PostgreSQL 部署**：不提供页面内 .db 下载/恢复，需由运维通过 mysqldump/pg_dump 等在服务器上备份与恢复；页面会给出相应提示。

## 六、功能边界与限制

以下为系统当前设计边界，便于正确使用与排错。

### 6.1 数据与业务

| 项目         | 说明 |
|--------------|------|
| 名单唯一标识 | 以**学号**唯一；同一学号重复导入时更新该条记录。 |
| 名单状态     | **生效**（status=1）、**已撤销**（status=0）；教师端仅能查询生效记录。 |
| 学号规则     | 长度 1～32 位（清洗后）；支持全角数字自动转半角。 |
| 工号         | 用户登录账号，长度限制 64 字符内。 |
| 密码         | 最少 6 位；建议使用强密码并定期修改。 |

### 6.2 文件与导入导出

| 项目           | 限制 |
|----------------|------|
| 批量导入 Excel | 必须包含列：姓名、学号、专业、原因、处分时间；支持 .xlsx、.xls。 |
| 单次导入行数   | 不超过 10,000 行。 |
| 单次上传文件大小 | 不超过 10 MB。 |
| 批量比对 Excel | 至少包含「学号」列；行数、文件大小与上述一致。 |
| 名单/审计导出  | 单次导出有最大条数限制（如 50,000 条），超出部分会提示。 |

### 6.3 会话与安全

| 项目         | 说明 |
|--------------|------|
| 会话超时     | 无操作约 30 分钟自动退出（具体以部署配置为准）。 |
| 登录失败冷却 | 同一账号连续失败约 5 次后，需等待约 5 分钟再试。 |
| 恢复操作     | 仅 SQLite 部署支持页面上传 .db 恢复；恢复前请确认备份并勾选确认项。 |

### 6.4 其他

- **多标签页**：建议同一账号不要在多标签页同时进行写操作（如导入、删除、恢复），以免结果不符合预期。
- **浏览器**：建议使用最新版本浏览器；若遇页面异常可尝试清除缓存或换用 Chrome/Edge。

## 七、技术栈说明（概要）

面向用户的技术概况，便于对外介绍或招标说明。

| 类别       | 技术/说明 |
|------------|-----------|
| 应用类型   | Web 应用（B/S），浏览器访问。 |
| 前端/界面  | Streamlit 构建的响应式 Web 界面。 |
| 后端语言   | Python 3.9+。 |
| 数据存储   | 关系型数据库：默认 SQLite；可选 MySQL、PostgreSQL（由部署配置决定）。 |
| 数据处理   | Pandas、OpenPyxl/xlrd 用于 Excel 解析与清洗。 |
| 安全       | 密码 BCrypt 加密；会话管理；审计日志；可选 HTTPS（由部署配置）。 |
| 图表展示   | Plotly 用于管理员仪表盘统计图表。 |
| 部署方式   | 支持单机部署（如办公室服务器）、云服务器部署；可选 Nginx 反向代理与 HTTPS。 |

系统采用分层架构，业务逻辑与数据访问分离，便于维护与扩展。

## 八、常见问题（FAQ）

**Q1：忘记密码怎么办？**  
由管理员在「用户管理」中为该账号重置密码。

**Q2：教师能否看到已撤销的名单？**  
不能。教师端仅能查询**生效中**的失信记录；已撤销记录仅管理员在「已撤销名单」中可见并可恢复为生效。

**Q3：批量比对 Excel 格式要求？**  
至少包含一列「学号」；列名需与系统要求一致（如批量导入需：姓名、学号、专业、原因、处分时间）。表头在第一行。

**Q4：导入时提示「学号为空」？**  
该行学号在清洗后为空，会被跳过；可在「上次导入结果」中查看跳过行并修正 Excel 后重新导入。

**Q5：页面提示「登录已超时」？**  
长时间未操作导致会话过期，请重新登录。

**Q6：能否在手机或平板上使用？**  
可以。使用浏览器访问系统地址即可；建议使用主流浏览器以获得较好体验。

**Q7：数据库备份在哪里？**  
SQLite 部署时，管理员可在「系统维护」中下载 .db 备份；同时建议运维定期在服务器上备份 `database.db` 与 `backups/` 目录。MySQL/PostgreSQL 需由运维使用数据库自带工具备份。
