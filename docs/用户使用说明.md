# 华中科技大学学术道德监督管理系统 — 用户使用说明

本文档面向最终用户，介绍系统功能、使用方式、功能边界及技术概况。

---

## 一、产品概述

### 1.1 系统简介

**华中科技大学学术道德监督管理系统**（Academic Dishonesty Management System）是一套面向高校的 Web 应用，用于：

- **名单管理**：维护校内学术失信/违规人员名单（生效中、已撤销），支持 PDF 认定结论上传与预览。
- **查询与比对**：教师按姓名/拼音/学号单条查询或上传 Excel 批量比对，判断学生是否在生效名单中。
- **审计与备份**：记录操作日志，支持数据备份与恢复（视部署方式而定）。

### 1.2 适用对象与场景

| 对象     | 典型场景 |
|----------|----------|
| 教务处/学工 | 导入名单、维护生效/撤销状态、查看统计、用户与系统维护 |
| 教师/辅导员 | 单条查询学生是否在名单中、批量比对名单（如答辩/评奖前）、浏览名单 |
| 系统管理员 | 用户账号、密码、备份与恢复、审计日志查看与导出 |

### 1.3 访问方式

- **系统地址**：由部署单位提供（如 http://服务器IP:8501）。
- **浏览器**：建议使用 Chrome、Edge、Safari 等现代浏览器。
- **网络**：需能访问部署该系统的服务器（校园网或公网视部署而定）。

## 二、角色与权限

系统有两种角色，权限边界清晰：

| 角色     | 权限概述 |
|----------|----------|
| **教师** | 仅可查询：单条查询、批量比对、只读浏览名单、查看个人操作记录；不可增删改名单与用户，不可导出。 |
| **管理员** | 名单的增删改、导入/导出、生效/撤销/恢复；PDF 管理；用户管理；审计日志；备份与恢复。 |

同一账号仅对应一种角色；登录后页面与侧栏功能按角色展示。

## 三、登录与安全

### 3.1 登录

- **工号**：登录账号（由管理员创建或使用默认管理员账号）。
- **密码**：首次使用请使用管理员告知的初始密码；管理员账号请在首次登录后修改默认密码。

### 3.2 安全机制

- **会话超时**：长时间无操作（默认约 30 分钟）将自动退出；超时前 5 分钟会显示倒计时提醒，需重新登录。
- **登录保护**：同一账号连续 5 次登录失败后，将进入 5 分钟冷却期。
- **水印**：页面带有当前登录人姓名与工号的水印，用于防截屏与责任追溯。
- **审计**：关键操作（登录、查询、导入、增删改、备份恢复、归档等）均会记录到审计日志。

### 3.3 退出登录

点击侧栏底部 **「退出登录」** 即可安全退出。

## 四、教师端功能说明

教师登录后，侧栏显示四个功能板块：单条查询、批量比对、名单查询、个人记录。

### 4.1 单条查询

- **入口**：侧栏选择「单条查询」。
- **操作**：输入学生**学号**、**姓名**、**姓名拼音全拼**（如 zhangwei）或 **姓名首字母**（如 zw）后查询。
- **多人查询**：可使用换行、逗号、顿号、分号分隔多个词；如需联合校验，可写成「姓名 + 学号」。
- **姓名空格兼容**：例如「张 伟」可命中「张伟」。
- **结果展示**：
  - **单条命中**：自动展开详情卡片，显示姓名、学号、单位、原因、日期，若有 PDF 认定结论可直接预览。
  - **多条命中**：先显示表格概览，每条记录下方有展开器可查看详情与 PDF。
  - **未命中**：提示「未查询到失信记录，所查学生信用良好」。

### 4.2 批量比对

- **入口**：侧栏选择「批量比对」。
- **操作**：上传 Excel 文件（.xlsx 或 .xls），表格中**至少包含一列「学号」**；可先下载模板。
- **结果**：
  - **有命中**：红色提示命中条数，表格展示命中记录（分页，默认每页 10 条），可下载比对结果 Excel。
  - **无命中**：绿色提示名单内学生信用良好。
- **说明**：学号会自动清洗（去空格、全角转半角等）；单次上传行数、文件大小有限制（见第六节）。

### 4.3 名单查询（只读）

- **入口**：侧栏选择「名单查询」。
- **功能**：浏览生效中的失信名单，支持姓名/拼音/学号搜索、单位（院系）筛选、排序、分页。
- **限制**：**不可导出**、**不可编辑**、**不可选择操作**。仅用于查阅和审核。
- **详情**：可点击展开详细信息与 PDF 预览。

### 4.4 个人记录

- **入口**：侧栏选择「个人记录」。
- **内容**：当前登录教师**最近的操作历史**（如单条查询、批量比对），最多展示 100 条；包含时间、类型、对象、简要详情。

## 五、管理员端功能说明

管理员登录后，侧栏有四个功能板块：名单查询、名单管理、系统维护、用户管理。

### 5.1 名单查询

- **搜索**：支持按姓名（中文/拼音全拼/首字母缩写）和学号搜索，支持多项输入。
- **单位筛选**：按院系分类树展开筛选。
- **列设置**：可自定义显示哪些列。
- **排序与分页**：可按各列排序，每页条数可调（10/20/50/100）。
- **详情卡片**：点击行展开详情，含 PDF 认定结论预览。
- **批量暂存**：勾选记录后可批量导出到 Excel。
- **全量导出**：可导出当前筛选条件下的所有记录。

### 5.2 名单管理

- **批量导入**
  - Excel 需包含列：**姓名、学号/工号、所在单位、处理原因、认定日期**。
  - 学号唯一：已存在则更新，不存在则新增；学号为空的行会跳过。
  - 支持 .xlsx、.xls；单次导入行数、文件大小有限制（见第六节）。

- **手动新增**
  - 单条录入：姓名、学号、单位、原因、日期；学号不可与已有记录重复。
  - 可同时上传 PDF 认定结论文件。

- **生效名单**
  - 支持按姓名、拼音、首字母、学号、单位筛选；分页、排序、每页条数可调。
  - 可对指定学号执行：软删除（设为已撤销）、编辑（含 PDF 替换）。

- **已撤销名单**
  - 同样支持筛选、分页、排序。
  - 可对指定学号执行：恢复为生效。

- **初始化名单**：将所有生效记录设为已撤销；需二次确认，请谨慎使用。

- **导出**：生效名单、已撤销名单均可按当前筛选与排序导出 Excel。

### 5.3 系统维护

- **审计日志**
  - 可按操作人、操作类型、日期筛选。
  - **导出机制**：先点击"准备导出数据"，待系统缓存后再点击下载，避免重复查询卡顿。
  - 支持导出 CSV 和 Excel 格式。
  - **归档清理**：可对历史日志进行归档（归档后审计记录从列表移除但保留归档审计记录）。

- **数据库备份与恢复**（视部署方式）
  - **自动备份告警**：若备份失败，页面顶部会出现红色告警。
  - **SQLite 部署**：可下载当前数据库 .db 文件；可上传备份 .db 恢复（需勾选确认）。
  - **PostgreSQL 部署**：不提供页面恢复，需运维使用 pg_dump 备份。

### 5.4 用户管理

- **用户列表**：查看所有用户（工号、姓名、角色、启用状态）。
- **新增用户**：填写工号（唯一）、姓名、密码、角色（管理员/教师）。
- **密码重置**：选择用户后重置密码。
- **启用/禁用**：禁用后该账号无法登录且已有会话立即失效；**不能禁用当前登录账号**。

## 六、功能边界与限制

### 6.1 数据与业务

| 项目         | 说明 |
|--------------|------|
| 名单唯一标识 | 以**学号**唯一；同一学号重复导入时更新该条记录。 |
| 名单状态     | **生效**（status=1）、**已撤销**（status=0）；教师端仅能查询生效记录。 |
| 学号规则     | 长度 1～32 位（清洗后）；支持全角数字自动转半角。 |
| 搜索结果上限 | 单次搜索最多返回 200 条候选结果。 |
| 密码         | 最少 6 位；建议使用强密码。 |

### 6.2 文件与导入导出

| 项目           | 限制 |
|----------------|------|
| 批量导入 Excel | 必须包含列：姓名、学号/工号、所在单位、处理原因、认定日期；支持 .xlsx、.xls。 |
| 单次导入行数   | 不超过 10,000 行。 |
| 单次上传文件大小 | 不超过 10 MB（Streamlit 层面同步限制）。 |
| PDF 上传       | 仅接受 PDF 格式，单文件不超过 10 MB，系统校验文件头魔数。 |
| 名单/审计导出  | 单次导出有最大条数限制（如 50,000 条）。 |

### 6.3 会话与安全

| 项目         | 说明 |
|--------------|------|
| 会话超时     | 无操作约 30 分钟自动退出（超时前 5 分钟有倒计时提醒）。 |
| 登录失败冷却 | 同一账号连续失败 5 次后，需等待 5 分钟再试。 |
| 恢复操作     | 仅 SQLite 部署支持页面恢复；恢复前请确认备份并勾选确认项。 |

### 6.4 其他

- **多标签页**：建议同一账号不要在多标签页同时进行写操作（如导入、删除、恢复），以免结果不符合预期。
- **浏览器**：建议使用最新版本浏览器；若遇页面异常可尝试清除缓存或换用 Chrome/Edge。

## 七、技术栈说明（概要）

| 类别       | 技术/说明 |
|------------|-----------| 
| 应用类型   | Web 应用（B/S），浏览器访问。 |
| 前端/界面  | Streamlit 构建的响应式 Web 界面。 |
| 后端语言   | Python 3.9+。 |
| 数据存储   | 关系型数据库：开发用 SQLite；生产推荐 PostgreSQL。 |
| 数据处理   | Pandas、OpenPyxl/xlrd 用于 Excel 解析与清洗。 |
| 搜索       | pypinyin 实现拼音全拼和首字母搜索。 |
| 安全       | 密码 BCrypt 加密；服务端会话管理；PDF 文件校验；路径穿越防护；Excel 导出防注入；审计日志。 |
| 部署方式   | 支持单机部署（办公室服务器）、云服务器部署；Nginx 反向代理 + 安全头 + IP 限流。 |

## 八、常见问题（FAQ）

**Q1：忘记密码怎么办？**
由管理员在「用户管理」中为该账号重置密码。

**Q2：教师能否看到已撤销的名单？**
不能。教师端仅能查询**生效中**的失信记录。

**Q3：教师能否导出名单？**
不能。教师端的「名单查询」功能为只读浏览，不提供导出按钮。

**Q4：支持拼音搜索吗？**
支持。可输入姓名的拼音全拼（如 zhangwei）或首字母缩写（如 zw）进行搜索。

**Q5：批量比对 Excel 格式要求？**
至少包含一列「学号」（或「学号/工号」）；表头在第一行。可先下载系统提供的模板。

**Q6：导入时提示「学号为空」？**
该行学号在清洗后为空，会被跳过；可修正 Excel 后重新导入。

**Q7：页面提示「登录已超时」？**
长时间未操作导致会话过期，请重新登录。

**Q8：能否在手机或平板上使用？**
可以。使用浏览器访问系统地址即可。

**Q9：PDF 预览看不到？**
请确认浏览器支持 PDF 内嵌显示（Chrome/Edge 均支持）。若使用手机浏览器可能需要单独下载查看。
