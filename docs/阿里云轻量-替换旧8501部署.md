# 方案 B：新建目录部署并替换原 8501 上的旧系统

在 **SSH** 下操作（阿里云轻量 → **远程连接**）。以下假设新目录为 **`/opt/hustsystem`**；若你希望用别的路径，全文替换即可。

---

## 一、先确认旧进程是谁在占 8501

```bash
sudo ss -tlnp | grep 8501
# 或
sudo lsof -i :8501
```

记下 **进程名 / 启动命令 / 工作目录**（常见：`streamlit`、`python`，或宝塔里的守护进程）。

---

## 二、停掉旧服务（按你机器上实际存在的方式执行）

### 2.1 若是 systemd（服务名可能是 `hustsystem`、`streamlit` 等）

```bash
systemctl list-units --type=service --all | grep -Ei 'hust|streamlit|8501'
```

若看到例如 `hustsystem.service`：

```bash
sudo systemctl stop hustsystem
sudo systemctl disable hustsystem
sudo rm -f /etc/systemd/system/hustsystem.service
sudo systemctl daemon-reload
```

（服务文件名以你机器上为准；不确定时先看：`ls /etc/systemd/system/*.service`）

### 2.2 若是宝塔「进程守护 / Supervisor」

登录 **宝塔面板** → 找到守护 **8501** 的项目 → **停止** 并 **删除** 该守护任务（避免和新 systemd 抢端口）。

### 2.3 若仍有进程占用 8501

```bash
sudo ss -tlnp | grep 8501
```

记下 **PID**，确认是旧业务后再结束（示例，**请把 PID 换成实际数字**）：

```bash
sudo kill PID
# 仍不退出再：sudo kill -9 PID
```

---

## 三、（可选）备份旧目录里的数据再删

若旧系统在项目目录下有 **`database.db`**、**`backups/`**，建议先打包：

```bash
# 把 /path/to/旧项目 换成你旧代码的真实路径
sudo tar czvf ~/hustsystem-old-backup-$(date +%Y%m%d).tgz -C /path/to/旧项目 database.db backups 2>/dev/null
```

确认新系统跑通、数据无误后，再手动删除旧目录（**路径务必核对**）：

```bash
# sudo rm -rf /path/to/旧项目
```

---

## 四、新建目录并部署新版本

```bash
sudo mkdir -p /opt/hustsystem
sudo chown "$USER:$USER" /opt/hustsystem
cd /opt/hustsystem
```

**拉代码**（二选一）：

```bash
# Git
git clone <你的仓库URL> .

# 或本机 scp 上传压缩包后在此解压
```

```bash
python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
python init_db.py
```

---

## 五、用 systemd 固定到新目录、仍监听 8501

创建服务（**把 `User=` 改成你的 Linux 用户名**，如 `root` 或 `www`）：

```bash
sudo nano /etc/systemd/system/hustsystem.service
```

写入：

```ini
[Unit]
Description=HUST 学术道德监督管理系统 (Streamlit)
After=network.target

[Service]
User=你的Linux用户名
WorkingDirectory=/opt/hustsystem
Environment="PATH=/opt/hustsystem/venv/bin"
ExecStart=/opt/hustsystem/venv/bin/python -m streamlit run app.py --server.address=0.0.0.0 --server.port=8501 --server.headless=true
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启用并启动：

```bash
sudo systemctl daemon-reload
sudo systemctl enable hustsystem
sudo systemctl start hustsystem
sudo systemctl status hustsystem
```

**Nginx** 若已是 `proxy_pass http://127.0.0.1:8501;`，**无需改配置**（仍反代到本机 8501）。

---

## 六、防火墙

轻量控制台 **防火墙** 放行 **22**；若用户走 **80** 访问，放行 **80**。调试直连 8501 时再临时放行 **8501**。

---

## 七、验证

```bash
curl -sI http://127.0.0.1:8501 | head -5
journalctl -u hustsystem -n 50 --no-pager
```

浏览器访问你的 **域名或 IP**（经 Nginx 则走 80；直连则 `http://公网IP:8501`）。

---

## 小结

| 步骤 | 动作 |
|------|------|
| 1 | 查占用 8501 的进程 / systemd / 宝塔守护 |
| 2 | `stop` + `disable` 旧服务，必要时 `kill`，宝塔里删掉旧守护 |
| 3 | 备份旧 `database.db` 等后，再删旧目录（可选） |
| 4 | 新建 `/opt/hustsystem`，装依赖、`init_db` |
| 5 | 新建 `hustsystem.service`，`WorkingDirectory` 指向新目录 |
| 6 | Nginx 仍指向 `127.0.0.1:8501` 则不用改 |

更多说明见 **`docs/阿里云ECS部署指南.md`**。
