# 华中科技大学学术道德监督管理系统 — 项目架构与模块说明

本文档面向开发者，详细梳理系统架构、模块职责、依赖关系与修改指南。

---

## 一、整体架构（分层）

```
┌─────────────────────────────────────────────────────────┐
│                    浏览器（用户）                          │
└────────────────────────┬────────────────────────────────┘
                         │ HTTP / WebSocket
┌────────────────────────▼────────────────────────────────┐
│                     app.py（应用入口）                     │
│  会话初始化 · 水印注入 · 路由分发 · 超时检查 · 自动备份     │
└──┬──────────┬──────────┬──────────┬─────────────────────┘
   │          │          │          │
   ▼          ▼          ▼          ▼
┌──────┐ ┌────────┐ ┌────────┐ ┌─────────────┐
│login │ │teacher │ │admin/  │ │ components  │
│.py   │ │(子模块)│ │(子模块)│ │ .py(通用UI) │
└──┬───┘ └──┬─────┘ └──┬─────┘ └──────┬──────┘
   │        │          │               │
   ▼        ▼          ▼               ▼
┌─────────────────────────────────────────────────────────┐
│                      core/（核心层）                      │
│  config · auth · search · session_store · database       │
│  models · student_id · excel_processor · file_safe_guard │
│  pdf_server · audit_logger · system_operations           │
└─────────────────────────┬───────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│              数据存储层（SQLite / PostgreSQL）              │
│              sessions.json · static/pdf_files/ · logs/    │
└─────────────────────────────────────────────────────────┘
```

**核心原则**：
- 上层依赖下层，下层不依赖上层
- `core/` 不 import `views/`；`views/` 不 import `app.py`
- 通用 UI 组件在 `views/components.py`，供所有视图共享

## 三、模块职责与接口

### 3.1 core/ — 核心层

| 模块 | 职责 | 对外接口 | 修改影响范围 |
|------|------|----------|-------------|
| **config.py** | 常量、文案、配置值 | 导出常量供全局使用 | 改文案/阈值：仅影响显示；改键名：需全局搜索替换 |
| **models.py** | 三张表的 ORM 定义 + 索引 | `User`、`Blacklist`、`AuditLog`、`Base` | 改字段：需同步改 views、Excel 映射 |
| **database.py** | 数据库连接与会话 | `engine`、`SessionLocal`、`db_session()`、`IS_SQLITE` | 改连接逻辑：影响所有数据库操作 |
| **auth.py** | 密码校验 | `verify_password(plain, hashed)` | 仅 login.py 调用 |
| **search.py** | 搜索引擎（拼音/多项） | `fetch_candidate_records()`、`fetch_teacher_candidate_records()` | 名单查询、单条查询均依赖 |
| **search_config.py** | 搜索配置 | `SEARCH_RESULT_HARD_LIMIT` 等 | search.py 依赖 |
| **session_store.py** | 服务端 Session | `create_session()`、`get_session()`、`delete_session()` | app.py、login.py 调用 |
| **student_id.py** | 学号清洗与校验 | `clean_student_id()`、`validate_student_id()` | Excel 导入、手动新增 |
| **excel_processor.py** | Excel 解析与导出 | `parse_blacklist_excel()`、`parse_batch_check_excel()`、`sanitize_for_export()` | 导入/导出/比对 |
| **file_safe_guard.py** | PDF 上传安全 | `validate_pdf_upload()`、`save_pdf_file()`、`remove_old_pdf()` | management.py |
| **pdf_server.py** | PDF 安全展示 | `render_pdf_preview()` | list_query.py、single_search.py |
| **audit_logger.py** | 审计日志写入 | `log_audit_action()` | 全局所有敏感操作 |
| **system_operations.py** | 备份/恢复 | `auto_backup()`、`get_db_file_bytes()` | app.py、system.py |

**core/ 内部依赖规则**：

```
config.py ← 无依赖（纯常量）
models.py ← sqlalchemy（纯结构）
database.py ← sqlalchemy, os, config
auth.py ← bcrypt
search.py ← database, models, search_config, pypinyin
search_config.py ← 无依赖（纯配置）
session_store.py ← config, 标准库
student_id.py ← config, pandas
excel_processor.py ← config, student_id, pandas
file_safe_guard.py ← config, 标准库
pdf_server.py ← config, streamlit
audit_logger.py ← database, models
system_operations.py ← database, config
```

## 五、核心调用链

### 5.1 请求生命周期

```
main()
 ├─ _init_session_state()          初始化 session 键
 ├─ _restore_session_from_sid()    从 URL sid 恢复登录态
 ├─ set_page_config()              页面配置
 ├─ _run_auto_backup_once()        首次备份（SQLite）
 ├─ _inject_watermark()            水印注入（html.escape）
 ├─ _render_sidebar()              侧栏渲染（含导航和登出）
 │
 ├─ 未登录 → render_login_page()
 │
 ├─ _check_session_timeout()       超时检查 + 5分钟预警
 │
 ├─ role=teacher → render_teacher_page()
 │   ├─ 单条查询 → render_single_search()
 │   ├─ 批量比对 → render_batch_check()
 │   ├─ 名单查询 → render_teacher_list_query(db)
 │   └─ 个人记录 → render_my_logs()
 │
 └─ role=admin → render_admin_page()
     └─ db_session() 内按导航分发：
         ├─ 名单查询 → _render_list_query(db)
         ├─ 名单管理 → _render_management(db)
         ├─ 系统维护 → _render_system(db)
         └─ 用户管理 → _render_user_management(db)
```

### 5.2 数据库会话管理

```
方式一（推荐）：with db_session() as db:
  └─ 上下文管理器自动 close

方式二（管理员页）：render_admin_page() 中统一开 db_session()
  └─ 传递 db 给子模块，避免嵌套多次开关
```

### 5.3 审计日志写入

```
所有需要记录的操作（登录、查询、导入、增删改、备份恢复、归档）
  └─ log_audit_action(action_type, target, details)
      └─ db_session() → AuditLog.add() → commit()
```

## 六、修改指南（按场景）

### 6.1 修改数据库字段

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `core/models.py` | 增/改/删 Column |
| 2 | `views/admin/management.py` | 同步修改导入映射和表单字段 |
| 3 | `views/components.py` | 同步修改表格渲染列和导出列 |
| 4 | `views/teacher/single_search.py` | 同步修改查询结果展示 |
| 5 | `core/excel_processor.py` | 若改了 Excel 必需列，同步修改 |
| 6 | 数据库 | 需要做数据迁移（ALTER TABLE 或重建） |

### 6.2 新增一个管理员功能板块

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `views/admin/` | 新建 `xxx.py`，定义 `_render_xxx(db)` |
| 2 | `views/admin/__init__.py` | 在 `ADMIN_NAV_OPTIONS` 中添加选项，在路由分支中添加 |
| 3 | 无需修改 `app.py` | 路由由 `admin/__init__.py` 统一管理 |

### 6.3 新增一个教师端功能

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `views/teacher/` | 新建 `xxx.py`，定义 `render_xxx()` |
| 2 | `views/teacher/__init__.py` | 导出新函数 |
| 3 | `views/teacher_page.py` | 在 `TEACHER_NAV_OPTIONS` 中添加选项，在路由分支中添加 |

### 6.4 修改搜索逻辑

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `core/search.py` | 修改召回/排序逻辑 |
| 2 | `core/search_config.py` | 若改阈值/权重 |
| 3 | 检查调用方 | `admin/list_query.py`、`teacher/single_search.py`、`components.py` |

### 6.5 修改配置项/阈值/文案

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `core/config.py` | 直接修改常量值 |
| 2 | 若改了常量名 | 全局搜索旧名并替换 |

## 七、安全边界（修改时不可破坏）

| 安全机制 | 涉及文件 | 说明 |
|----------|---------|------|
| 密码不明文存储 | auth.py、login.py、user_mgmt.py | 必须 BCrypt；日志中不记录密码 |
| DATABASE_URL 不暴露 | database.py | 不写入日志、不展示给前端 |
| 参数化查询 | components.py、search.py | 不可拼接 SQL 字符串；LIKE 必须转义 |
| 审计不可删减 | audit_logger.py | 只增不减，仅管理员可归档 |
| 水印 XSS 防护 | app.py | 水印内容必须 `html.escape()` |
| PDF 上传安全 | file_safe_guard.py | 魔数校验 + UUID 文件名 + 路径穿越防护 |
| 恢复仅 SQLite | system.py | PostgreSQL 禁止页面恢复 |
| 会话超时 | app.py | 不可绕过 |
| 登录冷却 | login.py | 不可削弱 |
| 禁用即踢下线 | user_mgmt.py | 禁用后清除所有会话 |
| Excel 导出防注入 | excel_processor.py | `sanitize_for_export()` |

## 八、性能关注点

| 场景 | 现有优化 | 文件 |
|------|---------|------|
| SQLite 并发 | WAL 模式 + synchronous=NORMAL | database.py |
| PostgreSQL 连接 | pool_size=10, pool_pre_ping, pool_recycle=3600 | database.py |
| 分页列表 | `.limit().offset()` 库内分页 | components.py |
| 搜索结果 | 硬上限 200 条 | search_config.py |
| 教师批量比对 | 学号分批 IN 查询（BATCH_SIZE=500） | batch_check.py |
| 审计日志导出 | 惰性缓存（先准备再下载） | system.py |
| 批量导入 | 分批 commit（BATCH_IMPORT_COMMIT_EVERY=100） | management.py |
| 导出 | 分批拉取 + 最大行数限制 | components.py |
| 索引 | Blacklist: (status,student_id), (status,name), 拼音字段；AuditLog: (timestamp) | models.py |
