# 华中科技大学学术道德监督管理系统 — 项目架构与模块说明

本文档面向开发者，详细梳理系统架构、模块职责、依赖关系与修改指南，方便后续大范围修改与优化时快速定位、降低引入 Bug 的风险。

---

## 一、整体架构（分层）

```
┌─────────────────────────────────────────────────────────┐
│                    浏览器（用户）                          │
└────────────────────────┬────────────────────────────────┘
                         │ HTTP / WebSocket
┌────────────────────────▼────────────────────────────────┐
│                     app.py（应用入口）                     │
│  会话初始化 · 水印注入 · 路由分发 · 超时检查 · 自动备份     │
└──┬──────────┬──────────┬──────────┬─────────────────────┘
   │          │          │          │
   ▼          ▼          ▼          ▼
┌──────┐ ┌────────┐ ┌────────┐ ┌─────────────┐
│login │ │teacher │ │admin/  │ │ components  │
│.py   │ │_page   │ │(子模块)│ │ .py(通用UI) │
└──┬───┘ └──┬─────┘ └──┬─────┘ └──────┬──────┘
   │        │          │               │
   ▼        ▼          ▼               ▼
┌─────────────────────────────────────────────────────────┐
│                      core/（核心层）                      │
│  config · auth · utils · session_store · database·models│
└─────────────────────────┬───────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────┐
│              数据存储层（database.db / MySQL / PG）        │
│              sessions.json · backups/ · logs/             │
└─────────────────────────────────────────────────────────┘
```

**核心原则**：  
- 上层依赖下层，下层不依赖上层  
- `core/` 不 import `views/`；`views/` 不 import `app.py`  
- 通用 UI 组件在 `views/components.py`，供所有视图共享

## 三、模块职责与接口

### 3.1 core/ — 核心层

| 模块 | 职责 | 对外接口 | 修改影响范围 |
|------|------|----------|------------|
| **config.py** | 常量、文案、配置值 | 导出常量供全局使用 | 改文案/阈值：仅影响显示或校验；改键名：需全局搜索替换 |
| **models.py** | 三张表的 ORM 定义 | `User`、`Blacklist`、`AuditLog`、`Base` | 改字段：需同步改 views 中的读写、Excel 导入映射、迁移脚本 |
| **database.py** | 数据库连接与会话 | `engine`、`SessionLocal`、`db_session()`、`IS_SQLITE`、`DATABASE_DIR` | 改连接逻辑：影响所有数据库操作 |
| **auth.py** | 密码校验 | `verify_password(plain, hashed)` | 仅 login.py 调用；改算法需同步改密码生成逻辑 |
| **session_store.py** | 服务端 Session | `create_session()`、`get_session()`、`delete_session()` | 仅 app.py 和 login.py 调用 |
| **utils.py** | 工具函数集合 | 学号处理、Excel 解析、备份、审计写入 | 被多个 views 调用；改签名需检查所有调用方 |

**core/ 内部依赖规则：**

```
config.py ← 无依赖（纯常量）
models.py ← sqlalchemy（纯结构）
database.py ← sqlalchemy, os
auth.py ← config, bcrypt
session_store.py ← config, 标准库
utils.py ← config, database, models, pandas, streamlit
```

### 3.2 views/ — 视图层

| 模块 | 职责 | 依赖的 core 模块 | 依赖的 views 模块 |
|------|------|-----------------|-----------------|
| **login.py** | 登录页渲染与校验 | auth, config, database, models, session_store, utils | 无 |
| **teacher_page.py** | 教师端三个功能 | config, database, models, utils | components |
| **admin_page.py** | 管理员入口代理 | 无 | admin/ |
| **components.py** | 通用 UI 组件 | config, models | 无 |
| **admin/\_\_init\_\_.py** | 管理员路由分发 | config, database | admin 子模块 |
| **admin/dashboard.py** | 仪表盘 | config, database, models | 无 |
| **admin/management.py** | 名单管理 | config, database, models, utils | components |
| **admin/system.py** | 系统维护 | config, database, models, utils | components |
| **admin/user_mgmt.py** | 用户管理 | config, models, utils | 无 |

## 五、核心调用链

### 5.1 请求生命周期

```
main()
 ├─ _init_session_state()          初始化 session 键
 ├─ _restore_session_from_sid()    从 URL sid 恢复登录态
 ├─ set_page_config()              页面配置
 ├─ _run_auto_backup_once()        首次备份（SQLite）
 ├─ _inject_watermark()            水印注入
 ├─ _render_sidebar()              侧栏渲染（含导航和登出）
 │
 ├─ 未登录 → render_login_page()
 │
 ├─ _check_session_timeout()       超时检查
 │
 ├─ role=teacher → render_teacher_page()
 │   ├─ 单条查询 → _render_single_search()
 │   ├─ 批量比对 → _render_batch_check()
 │   └─ 个人记录 → _render_my_logs()
 │
 └─ role=admin → render_admin_page()
     └─ db_session() 内按导航分发：
         ├─ 仪表盘   → _dashboard_fragment()
         ├─ 名单管理 → _render_management(db)
         ├─ 系统维护 → _render_system(db)
         └─ 用户管理 → _render_user_management(db)
```

### 5.2 数据库会话管理

```
方式一（推荐）：with db_session() as db:
  └─ 上下文管理器自动 close

方式二（管理员页）：render_admin_page() 中统一开 db_session()
  └─ 传递 db 给子模块，避免嵌套多次开关
```

### 5.3 审计日志写入

```
所有需要记录的操作（登录、查询、导入、增删改、备份恢复）
  └─ log_audit_action(action_type, target, details)
      └─ db_session() → AuditLog.add() → commit()
```

## 六、修改指南（按场景）

### 6.1 修改数据库字段

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `core/models.py` | 增/改/删 Column |
| 2 | `views/admin/management.py` | 同步修改导入映射（`_upsert_one_blacklist`）和表单字段 |
| 3 | `views/components.py` | 同步修改表格渲染列（`render_blacklist_table`）和导出列（`render_blacklist_export_button`） |
| 4 | `views/teacher_page.py` | 同步修改查询结果显示和批量比对报告列 |
| 5 | `core/utils.py` | 若改了 Excel 必需列，同步修改 `REQUIRED_EXCEL_COLUMNS` |
| 6 | `migrate_sqlite_to_external.py` | 同步修改迁移映射 |
| 7 | 数据库 | 需要做数据迁移（ALTER TABLE 或重建） |

### 6.2 新增一个管理员功能板块

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `views/admin/` | 新建 `xxx.py`，定义 `_render_xxx(db)` |
| 2 | `views/admin/__init__.py` | 在 `ADMIN_NAV_OPTIONS` 中添加选项，在 `render_admin_page()` 中添加分支 |
| 3 | 无需修改 `app.py` | 路由由 `admin/__init__.py` 统一管理 |

### 6.3 新增一个教师端功能

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `views/teacher_page.py` | 新增 `_render_xxx()` 函数 |
| 2 | `views/teacher_page.py` | 在 `TEACHER_NAV_OPTIONS` 中添加选项，在 `render_teacher_page()` 中添加分支 |
| 3 | 无需修改 `app.py` | 路由由 `teacher_page.py` 内部管理 |

### 6.4 修改通用 UI（分页、筛选、排序）

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `views/components.py` | 修改对应函数 |
| 2 | 检查调用方 | `management.py`、`system.py`、`teacher_page.py` 中使用了这些组件 |

### 6.5 修改登录/认证逻辑

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `views/login.py` | 修改登录流程 |
| 2 | `core/auth.py` | 若改密码算法，此处修改 |
| 3 | `core/session_store.py` | 若改 Session 存储方式（如 Redis） |
| 4 | `app.py` | 若改会话恢复/超时逻辑 |

### 6.6 修改配置项/阈值/文案

| 步骤 | 文件 | 操作 |
|------|------|------|
| 1 | `core/config.py` | 直接修改常量值 |
| 2 | 若改了常量名 | 全局搜索旧名并替换 |

## 八、安全边界（修改时不可破坏）

| 安全机制 | 涉及文件 | 说明 |
|----------|---------|------|
| 密码不明文存储 | auth.py、login.py、user_mgmt.py | 必须 BCrypt；日志中不记录密码 |
| DATABASE_URL 不暴露 | database.py、app.py | 不写入日志、不展示给前端 |
| 参数化查询 | components.py（LIKE 转义） | 不可拼接 SQL 字符串 |
| 审计不可删减 | utils.py（log_audit_action） | 只增不减，不提供删除接口 |
| 水印 XSS 防护 | app.py（html.escape） | 水印内容必须转义 |
| 恢复仅 SQLite | system.py（IS_SQLITE 判断） | MySQL/PG 禁止页面恢复 |
| 会话超时 | app.py（_check_session_timeout） | 不可绕过 |
| 登录冷却 | login.py（LOGIN_FAIL_MAX） | 不可削弱 |
| **路径遍历防护** | `utils.py`（safe_filename） | 上传的 PDF 文件名强制进入极严正则白名单清洗，彻底防御 `../` 或 `\0` 截断注入 |
| **存储泄漏防护** | `utils.py`（remove_old_pdf） | 替换原有黑名单 PDF 时，若原纪录存在旧 PDF，则安全静默删除旧磁盘文件，防止死数据占满服务器硬盘 |
| **备份告警机制** | `app.py` & `system.py` | 捕获备份异常（例如权限/磁盘满）通过 `st.session_state` 穿透至管理员维护页直接标红报错 |

## 九、性能关注点

| 场景 | 现有优化 | 文件 |
|------|---------|------|
| **高并发读写** | **启动 SQLite WAL (Write-Ahead Logging) 模式**，彻底解决超大批量 Excel 导入时界面卡死/数据库锁死的问题 | `core/database.py` |
| 仪表盘统计 | `@st.cache_data(ttl=60)` + 单次 GROUP BY | dashboard.py |
| 名单列表 | 库内分页+筛选+排序（不全量加载） | components.py |
| 教师批量比对 | 学号分批 IN 查询（BATCH_SIZE=500） | teacher_page.py |
| **审计日志导出** | **惰性缓存机制**：改为“点击准备 -> 放入缓存 -> 渲染下载按钮”，从源头上消除了频繁刷新带来的全表扫描灾难 | system.py |
| 导入 | 分批 commit（BATCH_IMPORT_COMMIT_EVERY=100） | management.py |
| 导出 | 分批拉取（EXPORT_BATCH_SIZE=2000）+最大行数限制 | components.py |
| 索引 | Blacklist (status, id_card), (status, name)；AuditLog (timestamp) | models.py |

## 十一、后续优化建议

### 11.1 可进一步模块化的地方

| 当前状态 | 建议 |
|---------|------|
| `teacher_page.py`（423 行）单文件包含三个功能 | 可拆分为 `views/teacher/` 子模块（类似 admin 的拆法） |
| `core/utils.py` 职责较多（学号+Excel+备份+审计） | 可拆分为 `core/student_id.py`、`core/excel.py`、`core/backup.py`，`log_audit_action` 留在 `utils.py` 或独立为 `core/audit.py` |
| `app.py` 中的会话清空逻辑重复（登出和超时都做一遍） | 可提取 `_clear_session()` 函数 |

### 11.2 修改前的检查清单

每次修改代码前，建议过一遍：

- [ ] 确认修改范围：用本文档「快速定位表」和「修改指南」确认涉及的文件
- [ ] 搜索调用方：全局搜索被修改函数/常量的引用
- [ ] 保持接口兼容：若改了函数签名，检查所有调用方
- [ ] 安全边界不破坏：对照第八节检查
- [ ] 测试关键流程：登录 → 查询 → 导入 → 导出 → 备份/恢复
