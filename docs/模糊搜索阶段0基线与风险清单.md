## 5. 风险清单（阶段 0 识别）

### R1（高）长期运行仍在 SQLite

- 现状：当前基线环境为 SQLite。
- 风险：数据量持续增长后，写入竞争和维护弹性不足。
- 建议：阶段 1 进入 PostgreSQL 路线并做迁移演练。

### R2（高）部分模糊流程为应用层遍历

- 现状：管理员模糊过滤中，存在先取结果再应用层匹配逻辑。
- 位置：[components.py](/E:/hustsystem/views/components.py#L83)
- 风险：数据规模上升时，查询时延与内存开销可能劣化。
- 建议：阶段 1 增加搜索辅助字段与索引，阶段 2 下推更多过滤到数据库层。

### R3（中）错字/近音查询未覆盖

- 现状：`张烜`、`王方` 无自动命中。
- 风险：用户可能误以为“模糊”应包含错字纠正。
- 建议：维持当前安全边界（建议候选而非自动命中），在阶段 2 明确 UI 提示和匹配解释。

### R4（中）长期备份不删除会增长磁盘占用

- 现状：已确认“数据库文件与备份长期保存，不自动删除”。
- 风险：桌面机磁盘容量逐步被占满。
- 建议：执行“本机热备 + 外置冷备 + 容量告警”策略（迁移归档，不删除）。
