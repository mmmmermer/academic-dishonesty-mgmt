## 通用前置准备

所有路线共享以下基础步骤：

```bash
# 1. 克隆仓库
git clone <本仓库地址> /opt/hustsystem
cd /opt/hustsystem

# 2. 创建虚拟环境
python3 -m venv venv
source venv/bin/activate    # Linux/Mac
# Windows: venv\Scripts\activate

# 3. 安装依赖
pip install -r requirements.txt

# 4. 初始化数据库（默认管理员 admin / 123456）
python init_db.py
```

> ⚠️ **安全提醒**：上线后 **务必立即修改默认管理员密码**。

## <a id="A"></a>路线 A：高机密局域网部署与硬件指南 (Intranet Vault)

在极高保密与一般并发（几人~十几人同频使用）的特定场景下，系统的防御痛点不是"扛压防崩"，而是**"防内鬼抓包、防物理失窃、收缩网络攻击面"**。

### 1. 硬件落地方案
* **方案 1（性价比/便携流）**：零刻 SER6 Pro 迷你主机或 Intel NUC，连接内网交换机，置于保险文件柜中。
* **方案 2（容灾冗余流，推荐）**：群晖 DS723+ 等企业级 NAS。底层启用 RAID1 双盘镜像。

### 2. 软件四重"铁桶防御"实施

1. **LUKS 全盘硬件层加密**：安装 Ubuntu 22.04 LTS 时勾选 `LVM with LUKS`。服务器即使连盘带机被盗走，无开机主密钥仍为废钢。
2. **切断外部倾听，转内网**：Streamlit 后端启动时强制使用 `--server.address 127.0.0.1`，并交由 Nginx 在本机处理。
3. **Nginx 前置 HTTPS（防御网线监听）**：哪怕在全可信的办公室内网，所有数据交互也必须由 Nginx 挂载 SSL（自签名证书即可）再传输。
4. **IP 白名单与软密码墙**：在 Nginx 的 `location /` 下设置 `allow 极小科室网段; deny all;`。

### 3. Windows 一键启停（局域网场景）

系统提供了 Windows 批处理脚本：

```batch
# 启动（默认使用 PostgreSQL, 端口 8501）
start_system.bat [数据库密码] [端口号]

# 停止
stop_system.bat [端口号]
```

## <a id="B"></a>路线 B：轻量级云服务器单机部署

适用于 2核 4G 以下云主机（如阿里云 ECS），拿开即跑。

### 1. 系统与依赖基础

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip git nginx
```

完成「通用前置准备」后继续：

### 2. 设置环境变量（SQLite 开发模式）

```bash
# 如果不使用 PostgreSQL，设置 SQLite 回退
export ALLOW_SQLITE_FALLBACK=1
```

### 3. Systemd 常驻运行

创建 `/etc/systemd/system/hustsystem.service`：
```ini
[Unit]
Description=学术失信管理系统 (Streamlit)
After=network.target

[Service]
User=www-data
WorkingDirectory=/opt/hustsystem
Environment="PATH=/opt/hustsystem/venv/bin"
Environment="ALLOW_SQLITE_FALLBACK=1"
ExecStart=/opt/hustsystem/venv/bin/python -m streamlit run app.py --server.address=127.0.0.1 --server.port=8501 --server.headless=true
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload && sudo systemctl enable hustsystem && sudo systemctl start hustsystem
```

### 4. Nginx 代理

使用项目提供的安全配置：

```bash
sudo cp deploy/nginx_hustsystem.conf /etc/nginx/sites-available/hustsystem
# 修改 server_name 为你的域名或公网 IP
sudo ln -s /etc/nginx/sites-available/hustsystem /etc/nginx/sites-enabled/
sudo systemctl reload nginx
```

该配置已包含：
- IP 级请求限流（5r/s，突发 30）
- 安全响应头（X-Frame-Options, X-Content-Type-Options, X-XSS-Protection）
- WebSocket 支持（Streamlit 必需）
- 上传大小限制
- HTTPS 可选配置段

## <a id="C"></a>路线 C：PostgreSQL 生产部署（推荐）

长期运行、数据持续增长的场景，推荐使用 PostgreSQL 替代 SQLite。

### 1. 安装 PostgreSQL

```bash
# 使用项目提供的初始化脚本
sudo bash deploy/setup_postgresql.sh
```

或手动：
```bash
sudo apt install -y postgresql postgresql-contrib
sudo -u postgres psql -c "CREATE USER hustuser WITH PASSWORD '你的密码';"
sudo -u postgres psql -c "CREATE DATABASE hustsystem OWNER hustuser ENCODING 'UTF8';"
```

### 2. 设置环境变量

```bash
export DATABASE_URL="postgresql+psycopg2://hustuser:你的密码@127.0.0.1:5432/hustsystem"
```

安装 Python 驱动：
```bash
pip install psycopg2-binary
```

### 3. 初始化并启动

```bash
python init_db.py
python -m streamlit run app.py --server.address=127.0.0.1 --server.port=8501 --server.headless=true
```

### 4. Systemd 服务

使用项目提供的服务文件：
```bash
sudo cp deploy/hustsystem_postgresql.service /etc/systemd/system/hustsystem.service
# 编辑文件，填入正确的 DATABASE_URL
sudo systemctl daemon-reload && sudo systemctl enable hustsystem && sudo systemctl start hustsystem
```

### 5. 数据库连接池配置

系统自动配置 PostgreSQL 连接池：
- `pool_size=10`：常驻连接数
- `pool_pre_ping=True`：连接健康检查
- `pool_recycle=3600`：每小时回收连接

### 6. 从 SQLite 迁移

如果已用 SQLite 积累了数据：
1. 确保旧的 `database.db` 在项目根目录；
2. 设置好 `DATABASE_URL` 环境变量；
3. 执行：`python migrate_sqlite_to_external.py`

> **注意**：转入 PostgreSQL 后，前端的"数据恢复"功能会自动停用，需使用 `pg_dump` 备份。

## 备份策略

### SQLite 备份

- **自动备份**：系统每天首次访问时自动备份到 `backups/` 目录。
- **手动备份**：管理员可在「系统维护」中下载 `.db` 文件。
- **备份告警**：若自动备份失败（磁盘满/权限问题），管理员页面会显示红色告警。

### PostgreSQL 备份

使用项目提供的备份脚本模板：
```bash
# 参考 deploy/backup_example.sh
pg_dump -U hustuser hustsystem > /opt/backups/hustsystem_$(date +%Y%m%d_%H%M%S).sql
```

建议设置 crontab 定时任务：
```bash
0 2 * * * /opt/hustsystem/deploy/backup_example.sh
```

## Nginx 配置详解

项目提供了生产级 Nginx 配置 `deploy/nginx_hustsystem.conf`，包括：

| 特性 | 说明 |
|------|------|
| IP 请求限流 | `limit_req_zone` 5r/s per IP，burst=30 |
| 安全响应头 | X-Frame-Options, X-Content-Type-Options, X-XSS-Protection, Referrer-Policy |
| 上传限制 | `client_max_body_size 15m` |
| WebSocket | `proxy_set_header Upgrade/Connection`（Streamlit 必需） |
| 会话超时 | `proxy_read_timeout 86400` |
| HTTPS | 预留配置段，取消注释后配合 Certbot 使用 |

## 🆘 常见问题集锦 FAQ

**Q1：为何访问 Nginx 给出的 80 网页一直白屏或报错"Please wait..."？**
> 90% 因为 Streamlit WebSocket 通信被 Nginx 掐断。请检查 Nginx 中是否保留了 `proxy_set_header Upgrade $http_upgrade; proxy_set_header Connection "upgrade";` 核心指令，并赋予足够的 `proxy_read_timeout`。

**Q2：上传 PDF 或 Excel 报 `Payload Too Large`？**
> 系统已通过 `.streamlit/config.toml` 限制 10MB。若 Nginx 拦截，请检查 `client_max_body_size`。

**Q3：PostgreSQL 连接后仍使用 SQLite？**
> 检查 `DATABASE_URL` 环境变量是否正确设置。在 systemd 中需在 `[Service]` 段设置 `Environment="DATABASE_URL=..."`。

**Q4：多实例负载均衡时会话频繁登出？**
> Streamlit 的 WebSocket 是有状态的。多实例时 Nginx Upstream 必须加 `ip_hash` 确保同一浏览器连接到同一 Worker。
