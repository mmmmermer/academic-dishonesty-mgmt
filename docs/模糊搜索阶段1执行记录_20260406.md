## 10. 后台启停脚本（单机/桌面机稳定运行）

新增脚本：

- `scripts/start_app_postgres_bg.ps1`：后台启动 PostgreSQL 模式应用，写入 PID 文件与日志文件。
- `scripts/stop_app_postgres_bg.ps1`：按 PID 与端口监听进程双重清理，保证端口释放。

执行样例：

```powershell
powershell -ExecutionPolicy Bypass -File E:\hustsystem\scripts\start_app_postgres_bg.ps1 -DbPassword 123456 -AppPort 8501
powershell -ExecutionPolicy Bypass -File E:\hustsystem\scripts\stop_app_postgres_bg.ps1 -AppPort 8501
```

运行保障要点：

- 启动脚本会检测端口冲突并拒绝覆盖运行。
- 启动后写入 `tmp/streamlit_pg_<port>.pid`，日志写入 `logs/streamlit_pg_<port>.out.log` 与 `logs/streamlit_pg_<port>.err.log`。
- 停止脚本若发现遗留监听进程，会继续按端口清理，避免“看似停止但端口仍被占用”。
- 即使 PID 文件丢失，停止脚本也可按端口回收进程，适配历史手工启动场景。
