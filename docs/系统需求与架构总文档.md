## 一、系统概述与需求来源

**系统名称**：华中科技大学学术道德监督管理系统（Academic Dishonesty Management System）

**业务定位**：
- 用于高校内部管理的 Web 系统，记录、公示和查询学术失信人员名单。
- 核心逻辑：Excel 导入名单库 → 数据清洗（学号标准化）→ 数据库存储 → 前端公示与比对查询。
- 身份标识：名单以**工号/学号**（字段名 `student_id`，历史别名 `id_card`）为唯一标识。
- 部署环境：学校办公室局域网（Windows/Linux Server）。

**技术特点**：
- 轻量级部署：支持 SQLite 单文件数据库（开发/演示）；生产推荐 PostgreSQL。
- 简单易用：Streamlit 框架，快速构建 Web 界面。
- 安全可靠：BCrypt 密码加密、服务端会话管理、审计日志、PDF 文件校验、路径穿越防护。
- 角色权限：教师（只读查询）和管理员（全功能管理）。
- 高级搜索：支持中文姓名、拼音全拼、首字母缩写、学号、多项混合输入。

## 二、需求与功能

### 2.1 教师端（只读）

- **单条查询**：输入姓名/拼音/首字母/学号（支持换行、逗号、分号分隔多项）；仅查询 `status=1` 的生效记录；单条结果自动展开详情卡片含 PDF 预览；多条结果以表格+展开器形式呈现。
- **批量比对**：上传 Excel（.xlsx/.xls），至少包含「学号」列；学号清洗后与生效名单按批 500 分批 IN 查询。有命中时分页展示（默认每页 10 条）+ 下载比对结果报告 Excel；无命中时绿色成功提示。
- **名单查询**（只读）：与管理员端名单查询功能一致，但**禁止导出**、**禁止编辑**、**禁止选择操作**，仅提供搜索、分页、排序、详情卡片与 PDF 预览。
- **个人记录**：当前教师最近 100 条操作历史。

### 2.2 管理员端

- **名单查询**：沉浸式搜索面板，支持姓名/拼音/首字母/学号筛选 + 单位（院系）分类树筛选 + 列设置 + 排序 + 分页 + 批量暂存选择 + 全量导出。点击行可展开详情卡片含 PDF 预览。
- **名单管理**：批量导入（必需列：姓名、学号/工号、所在单位、处理原因、认定日期；学号唯一，已存在则更新）；单条新增（含 PDF 认定结论上传）；生效/已撤销列表（筛选、排序、分页、软删除、编辑、PDF 管理）；初始化名单（二次确认）。
- **系统维护**：审计日志（筛选、分页、导出 CSV/Excel、归档清理）；数据库备份/恢复（SQLite 页面操作，PostgreSQL 提示使用 pg_dump）。
- **用户管理**：用户列表、新增用户、密码重置、启用/禁用（不可禁用当前登录账号，禁用后立即清除该用户所有会话）。

### 2.3 UI/UX 与约束

- 水印防护：全局背景浅灰色水印「当前登录人姓名 + 工号」，`html.escape()` 防 XSS。
- 交互反馈：耗时操作使用 `st.spinner`；容错通过 `st.error` 等展示中文提示，不暴露 Traceback。
- Excel：.xlsx 用 openpyxl、.xls 用 xlrd；导出使用 `sanitize_for_export()` 防公式注入。
- 侧边栏导航：管理员四板块（名单查询/名单管理/系统维护/用户管理）；教师四板块（单条查询/批量比对/名单查询/个人记录）。
- 会话超时预警：超时前 5 分钟显示倒计时提醒。

## 三、技术栈与架构

| 层级       | 技术/工具        | 用途说明 |
|------------|------------------|----------|
| Web 框架   | Streamlit ≥1.54  | UI 构建、状态管理（st.session_state） |
| 数据库     | SQLite / PostgreSQL | 单文件（开发）或外部库（生产），由 `DATABASE_URL` 决定 |
| ORM        | SQLAlchemy ≥2.0  | 数据库交互、模型定义、连接池 |
| 数据处理   | Pandas ≥2.3、OpenPyxl/xlrd | Excel 读取与清洗 |
| 安全       | BCrypt ≥5.0      | 密码哈希加密与校验 |
| 搜索       | pypinyin ≥0.55   | 中文拼音转换（全拼/首字母） |
| 语言       | Python 3.9+      | 核心开发语言 |

核心业务流程：`Excel 导入名单 → 数据清洗（学号标准化）→ 数据库存储 → 前端公示与比对查询`

架构分层：
```
表现层（views/）→ 应用控制层（app.py）→ 业务逻辑层（core/）→ 数据存储层（DB + 文件系统）
```

## 四、数据模型

- **users**：id, username（工号唯一）, password_hash（BCrypt）, full_name, role（'admin'/'teacher'）, is_active。
- **blacklist**：id, name, student_id（字段别名 id_card）, major, reason, punishment_date, pdf_file_path, status（1=生效/0=已撤销）, name_pinyin_full, name_pinyin_abbr, created_at。
- **audit_logs**：id, operator_username, operator_full_name, action_type, target, details, timestamp。只增不减。

数据关系：用户与审计日志通过 operator_username 关联；名单与审计通过 action_type/target 间接关联。

**索引**：
- `blacklist`: `(status, student_id)`, `(status, name)`, `(status, name_pinyin_full)`, `(status, name_pinyin_abbr)`
- `audit_logs`: `(timestamp)`

## 五、会话与认证

- 双重会话：客户端 `st.session_state` + 服务端 `sessions.json`，通过 URL 参数 `sid` 恢复刷新后的登录态。
- 安全机制：BCrypt 存储、登录失败限制（5 次后冷却 5 分钟）、会话超时（30 分钟）、超时前 5 分钟预警、水印、审计日志。
- 令牌强度：`secrets.token_urlsafe(32)`（256 位熵）。
- 并发保护：`sessions.json` 使用 `threading.RLock` + 平台文件锁（Linux: `fcntl`, Windows: `msvcrt`）。

## 六、搜索引擎

搜索模块 `core/search.py` 实现了分层召回策略：

1. **精确层**：学号精确匹配、姓名精确匹配
2. **可控模糊层**：
   - 拼音全拼匹配（`zhangwei` → 张伟）
   - 首字母缩写匹配（`zw` → 张伟）
   - 姓名前缀/子串匹配
3. **多项输入**：支持换行、逗号、顿号、分号分隔多个查询词

配置项集中在 `core/search_config.py`，包括硬上限 `SEARCH_RESULT_HARD_LIMIT = 200`。

## 七、部署与运维

### 7.1 单机部署（办公室服务器）

- 环境：Python 3.9+，`pip install -r requirements.txt`。
- 首次执行 `python init_db.py`（默认管理员 admin/123456），**上线后务必修改默认密码**。
- 监听：`python -m streamlit run app.py --server.address=0.0.0.0 --server.port=8501`。
- Windows 一键启动：`start_system.bat [密码] [端口]`；一键停止：`stop_system.bat [端口]`。

### 7.2 使用 PostgreSQL（生产推荐）

- 设置 `DATABASE_URL`（如 `postgresql+psycopg2://用户:密码@主机:5432/库名`）。
- 安装 `psycopg2-binary`。
- 连接池：`pool_size=10, pool_pre_ping=True, pool_recycle=3600`。

### 7.3 Nginx 反向代理

配置文件：`deploy/nginx_hustsystem.conf`，包含：
- IP 级请求限流（5r/s，突发 30）
- 安全响应头（X-Frame-Options, X-Content-Type-Options, X-XSS-Protection, Referrer-Policy）
- 上传大小限制（15M）
- WebSocket 支持（Streamlit 必需）
- HTTPS 可选配置段

### 7.4 部署目录（deploy/）

| 文件 | 说明 |
|------|------|
| nginx_hustsystem.conf | Nginx 生产配置：安全头 + 限流 + WebSocket |
| nginx_example.conf | 多实例 Nginx 配置模板 |
| hustsystem.service.example | systemd 服务配置示例（SQLite） |
| hustsystem_postgresql.service | systemd 服务配置示例（PostgreSQL） |
| setup_postgresql.sh | PostgreSQL 初始化脚本 |
| deploy_sqlite.sh | SQLite 部署脚本 |
| backup_example.sh | 生产备份示例（pg_dump） |
| start_streamlit_workers.sh | 多进程启动脚本 |

## 八、安全机制汇总

| 安全机制 | 实现 |
|----------|------|
| 密码存储 | BCrypt + 72字节上限 |
| 会话令牌 | `secrets.token_urlsafe(32)` |
| 暴力破解防护 | 5次失败 + 5分钟冷却 |
| SQL 注入防护 | 全部 ORM 参数化 + LIKE 转义 |
| PDF 上传校验 | `%PDF` 魔数 + 10MB 限制 + UUID 文件名 |
| 路径穿越防护 | `abs_path.startswith(abs_pdf_dir + os.sep)` |
| Excel 导出防注入 | `=+\-@` 前缀加 `'` 转义 |
| XSS 防护 | 水印 `html.escape()` |
| 错误堆栈不暴露 | 生产仅友好消息 + 日志记录 |
| 禁用账号 | 禁用后立即清除所有会话 |
| 爬虫防护 | `robots.txt` → `Disallow: /` |
| Nginx | 安全头 + IP 限流 + HTTPS 可选 |

## 九、项目结构

```
hustsystem/
├── app.py                      # 应用入口（路由、会话、水印、超时）
├── init_db.py                  # 数据库初始化脚本
├── requirements.txt            # Python 依赖
├── .env.example                # 环境变量配置示例
├── .cursorrules                # AI 协作开发规则
├── start_system.bat / stop_system.bat  # Windows 启停脚本
│
├── core/                       # 核心层（14 个模块）
│   ├── config.py               # 常量、文案、阈值
│   ├── models.py               # ORM 模型 + 索引
│   ├── database.py             # 连接池、db_session()
│   ├── auth.py                 # BCrypt 密码校验
│   ├── search.py               # 搜索引擎（拼音/首字母/多项）
│   ├── search_config.py        # 搜索配置（权重/阈值/上限）
│   ├── session_store.py        # 服务端会话（JSON + 文件锁）
│   ├── student_id.py           # 学号清洗与校验
│   ├── excel_processor.py      # Excel 导入/导出/防注入
│   ├── file_safe_guard.py      # PDF 上传校验/路径穿越防护
│   ├── pdf_server.py           # PDF 安全提供服务
│   ├── audit_logger.py         # 审计日志写入
│   └── system_operations.py    # 数据库备份/恢复
│
├── views/                      # 视图层
│   ├── login.py                # 登录页
│   ├── components.py           # 通用 UI（表格/分页/筛选/导出）
│   ├── admin_page.py           # 管理员入口代理
│   ├── teacher_page.py         # 教师端路由器
│   ├── admin/                  # 管理员子模块（4 文件）
│   └── teacher/                # 教师端子模块（4 文件）
│
├── docs/                       # 项目文档
├── deploy/                     # 部署配置与脚本
├── scripts/                    # 运维 PowerShell 脚本
└── static/                     # 静态文件（robots.txt, pdf_files/）
```
