# 华中科技大学学术道德监督管理系统 — UI 设计布局与交互说明

> 本文档描述当前系统（Streamlit 实现）的页面结构、组件层级与交互行为，便于复制到其他 UI 设计工具（如 Figma、即时设计、AI 设计助手等）进行重新设计。

---

## 一、全局结构

### 1.1 页面配置
- **布局**：宽屏（`layout="wide"`）
- **页面标题**：学术失信人员管理系统
- **路由逻辑**：
  - 未登录 → 仅显示登录页
  - 已登录 + 教师角色 → 教师端主内容
  - 已登录 + 管理员角色 → 管理员端主内容
  - 会话超时或未知角色 → 提示后回到未登录状态

### 1.2 侧边栏（Sidebar）
- **显示条件**：始终存在，内容随登录状态变化
- **登录后内容**：
  - 文案：**当前用户**：{姓名}（{工号}）
  - 文案：**角色**：管理员 / 教师
  - 分割线
  - **功能导航**：Radio 按钮组（管理员 4 项 / 教师 4 项）
  - 按钮：「退出登录」— 点击后清除会话并刷新为登录页
- **未登录时**：侧边栏留空

### 1.3 全局水印（仅登录后）
- **位置**：全屏固定覆盖（fixed，top/left 0，宽高 100%）
- **内容**：{姓名} {工号}，单行大号文字
- **样式**：半透明（约 14% opacity）、灰色、旋转约 -25°、居中显示
- **交互**：不响应点击（pointer-events: none），不阻挡操作
- **层级**：最高（z-index 最大）

### 1.4 会话超时
- 无操作超过 30 分钟后自动退出
- **超时前 5 分钟**：显示 `st.warning` 倒计时提醒

## 二、登录页

### 2.1 层级结构
```
页面标题（一级标题）：华中科技大学学术道德监督管理系统
副标题：请登录

[登录表单]
  用户名（工号）[单行文本]
  密码 [密码框]
  [登录] 按钮

（表单下方）错误/成功消息区域
```

### 2.2 交互与反馈
- 未输入：提示「请输入用户名和密码。」
- 用户名或密码错误：提示「用户名或密码错误。」
- 账号已停用：提示「该账号已停用，请联系管理员。」
- 连续失败过多：提示「登录失败次数过多，请 5 分钟后再试。」
- 校验通过：写入会话、跳转到对应角色主页面

## 三、教师端

### 3.1 主框架
- **一级标题**：🎓 学术诚信档案查询
- **说明文案**：查询生效中的失信记录，学号完整显示。
- **侧栏 Radio 导航**（四项）：
  1. › 单条查询
  2. › 批量比对
  3. › 名单查询
  4. › 个人记录

---

### 3.2 单条查询

**布局：**
```
说明：输入学生的姓名、拼音、首字母或学号进行查询。

[文本域]
  placeholder：输入姓名/拼音/学号，多项可用换行、逗号、分号分隔

[🔍 查询] 按钮
```

**结果展示方式：**

- **单条命中 → 自动展开详情卡片**：
  ```
  ⚠️ 查询到失信/违规记录，请核实。
  
  [详情卡片] — 自动展开
    📌 姓名 — 学号
    ├─ 所在单位：xxx
    ├─ 处理原因：xxx
    ├─ 认定日期：xxx
    └─ [PDF 预览区域]（若有认定结论 PDF，以 iframe 嵌入预览）
  ```

- **多条命中 → 表格 + 逐条展开器**：
  ```
  ⚠️ 查到 N 条记录。
  
  [表格] 姓名、学号、所在单位、处理原因、认定日期
  
  [展开器: 姓名 — 学号]
    （同详情卡片内容 + PDF 预览）
  [展开器: 姓名 — 学号]
    ...
  ```

- **无命中**：成功提示「✅ 未查询到失信记录，所查学生信用良好。」

### 3.3 批量比对

**布局：**
```
副标题：批量比对
说明：上传含「学号」列的 Excel，系统会与当前生效名单比对。

[下载模板] 按钮
[文件上传] 选择 Excel 文件（.xlsx / .xls）
[开始比对] 按钮

（有命中时）
⚠️ 共命中 N 条失信/违规记录，请核实。（当前第 x/y 页）
[表格] 列：姓名、学号、所在单位、处理原因、认定日期（分页每页 10 条）
[分页区域] 上一页 | 第x页/共y页 | 下一页
[下载比对结果报告 (Excel)] 按钮

（无命中时）✅ 未查到违规记录，名单内学生信用良好。
```

---

### 3.4 名单查询（只读）

**布局：**
```
副标题：名单查询

[搜索控件]
  姓名/拼音/学号 [文本输入]
  所在单位 [下拉筛选]

[表格] 列：姓名、学号、所在单位、处理原因、认定日期
  — 无选择框（禁止勾选）
  — 无导出按钮
  — 无编辑操作

[分页] 简易分页（上一页/下一页/跳转）

（点击行展开）
[详情卡片] + PDF 预览（若有）
```

**关键限制**：
- 不渲染复选框（无 selection_key）
- 不渲染导出按钮
- 不提供编辑/删除操作
- 仅展示 status=1（生效）记录

### 4.2 名单查询

沉浸式搜索面板，提供全量名单检索与详情查看。

**布局：**
```
[搜索控件行] 三列
  姓名/拼音/学号 [文本输入] | 所在单位 [下拉筛选] | 状态 [下拉：生效/已撤销]

[列设置] 可选择显示哪些列
[排序] 可选择按哪一列排序 + 升序/降序
[每页条数] 下拉：10/20/50/100

[表格] 可勾选行（带复选框）
  列：姓名、学号、所在单位、处理原因、认定日期 等

[分页区域] 上一页 | 页码跳转 | 下一页 | 共N条

（表格下方）
[暂存选中] 已选 M 条记录
[导出选中 Excel] 按钮
[导出全部 (当前筛选) Excel] 按钮

（点击行展开）
[详情卡片]
  📌 姓名 — 学号
  ├─ 所在单位
  ├─ 处理原因
  ├─ 认定日期
  └─ [PDF 预览 iframe]（若有认定结论 PDF）
```

### 4.3 名单管理

长页面，自上而下分为多个区块。

#### 4.3.1 批量导入

```
副标题：一、录入与导入
说明：从 Excel 批量写入或更新名单。

[文件上传] 上传 Excel (.xlsx / .xls)
（上传解析后）预览前 10 行 [表格]
[开始导入] 按钮

导入说明：学号唯一，已存在则更新；学号为空行跳过。
```

#### 4.3.2 手动新增

```
副标题：手动新增

[表单]
  姓名 [单行]
  学号 [单行]
  所在单位 [单行]
  处理原因 [多行]
  认定日期 [日期选择]
  认定结论 PDF [文件上传，可选]
  [添加] 按钮
```

#### 4.3.3 生效名单

```
副标题：二、查看与维护（生效名单）

[筛选行] 姓名/拼音/学号 | 所在单位 | 排序 | 每页条数
[表格] 分页展示
[导出当前筛选 Excel] 按钮

按学号操作：
  输入学号 → [软删除] 按钮
  输入学号 → [编辑] 按钮
    → 展开编辑表单（预填，学号只读，可替换 PDF）
```

#### 4.3.4 已撤销名单

```
副标题：三、已撤销名单

（同生效名单结构）
按学号操作：输入学号 → [恢复为生效] 按钮
```

#### 4.3.5 初始化名单

```
副标题：初始化名单
警告说明 + 二次确认
```

## 五、通用组件与交互模式汇总

| 模式 | 说明 |
|------|------|
| **导航** | 侧栏 Radio 按钮组，管理员 4 项、教师 4 项 |
| **详情卡片** | 点击行/展开器后显示完整记录信息 + PDF 预览（iframe） |
| **表单** | 登录、手动新增、编辑、新增用户等；提交后校验，失败用错误提示 |
| **二次确认** | 初始化名单、数据恢复：先警告再确认 |
| **分页** | 简易分页（上一页/下一页）或完整分页（含跳转），视模块而定 |
| **筛选** | 文本输入（支持拼音/首字母）、下拉选择（单位、操作类型等） |
| **文件上传** | Excel（.xlsx/.xls）、PDF、数据库备份（.db） |
| **下载按钮** | Excel 导出、CSV 导出、数据库 .db 下载、比对结果报告 |
| **表格** | `st.dataframe` 占满容器宽度；列名见各区块说明 |
| **加载状态** | `st.spinner` 显示操作进行中 |
| **反馈类型** | 成功（绿）、错误（红）、警告（黄）、信息（蓝）；重要操作成功可配合气球动效 |
| **PDF 预览** | 通过 `st.components.v1.iframe` 嵌入 PDF 文件的 base64 数据 |

## 七、技术栈与实现备注

- 当前实现为 **Streamlit**，单页应用；侧边栏 Radio 按钮导航替代 Tab 切换。
- 表格为 `st.dataframe`，支持横向滚动；筛选通过独立控件完成。
- 详情卡片使用 `st.expander` 或 `st.container` + `st.markdown` 构建。
- PDF 预览使用 `st.components.v1.iframe` + base64 编码的 PDF 数据。
- 水印为 HTML/CSS 注入，全屏固定、居中、旋转、低透明度。
