# 华中科技大学学术道德监督管理系统 — UI 设计布局与交互说明

> 本文档描述当前系统（Streamlit 实现）的页面结构、组件层级与交互行为，便于复制到其他 UI 设计工具（如 Figma、即时设计、AI 设计助手等）进行重新设计。

---

## 一、全局结构

### 1.1 页面配置
- **布局**：宽屏（`layout="wide"`）
- **页面标题**：学术失信人员管理系统
- **路由逻辑**：
  - 未登录 → 仅显示登录页
  - 已登录 + 教师角色 → 教师端主内容
  - 已登录 + 管理员角色 → 管理员端主内容
  - 会话超时或未知角色 → 提示后回到未登录状态

### 1.2 侧边栏（Sidebar）
- **显示条件**：始终存在，内容随登录状态变化
- **登录后内容**：
  - 文案：**当前用户**：{姓名}（{工号}）
  - 文案：**角色**：管理员 / 教师
  - 分割线
  - 按钮：「退出登录」— 点击后清除会话并刷新为登录页
- **未登录时**：侧边栏可留空或仅保留分割线

### 1.3 全局水印（仅登录后）
- **位置**：全屏固定覆盖（fixed，top/left 0，宽高 100%）
- **内容**：{姓名} {工号}，单行大号文字
- **样式**：半透明（约 14% opacity）、灰色、旋转约 -25°、居中显示
- **交互**：不响应点击（pointer-events: none），不阻挡操作
- **层级**：最高（z-index 最大），登录后全局可见，退出登录后移除

### 1.4 会话超时
- 无操作超过设定时间（如 30 分钟）后，再次操作会提示「登录已超时，请重新登录。」并清除登录状态，主内容区显示登录页。

## 二、登录页

### 2.1 层级结构
```
页面标题（一级标题）：华中科技大学学术道德监督管理系统
副标题：请登录

[登录表单]
  用户名（工号）[单行文本]
  密码 [密码框]
  [登录] 按钮

（表单下方）错误/成功消息区域（如 st.error / st.success）
```

### 2.2 组件说明
| 组件 | 类型 | 说明 |
|------|------|------|
| 页面标题 | 一级标题 | 固定文案 |
| 副标题 | 二级标题/说明 | 「请登录」 |
| 用户名 | 单行输入 | 标签「用户名（工号）」 |
| 密码 | 密码输入 | 标签「密码」 |
| 登录 | 表单内主按钮 | 提交后校验，失败显示错误信息 |

### 2.3 交互与反馈
- 提交时：若用户名为空或密码为空 → 提示「请输入用户名和密码。」
- 用户名或密码错误 → 提示「用户名或密码错误。」
- 账号已停用 → 提示「该账号已停用，请联系管理员。」
- 同一账号连续失败次数过多且在冷却期内 → 提示「登录失败次数过多，请 5 分钟后再试。」
- 校验通过 → 写入会话、跳转到对应角色主页面（教师/管理员）。

## 三、教师端

### 3.1 主框架
- **一级标题**：🎓 学术诚信档案查询
- **说明文案**：查询生效中的失信记录，学号完整显示。
- **Tab 导航**（三个 Tab，从左到右）：
  1. 🔍 查单个学生
  2. 📤 批量查名单
  3. 📋 我的操作记录

---

### 3.2 Tab1：查单个学生

**布局：**
```
说明：输入一名学生的姓名或学号，查看是否在生效的失信名单中。

[表单]
  姓名或学号 [单行文本，placeholder：输入后按回车或点击查询]
  [🔍 查询] 按钮

（提交后下方动态内容）
```

**提交后反馈：**
- 未输入：提示「请输入姓名或学号后再查询。」
- 查询中：显示「正在查询...」加载状态
- 无命中：成功提示「✅ 未查询到违规记录，该生信用良好。」
- 有命中：警告提示「⚠️ 查询到失信/违规记录，请核实。」+ **表格**  
  表格列：姓名、学号、专业、原因、处分日期

---

### 3.3 Tab2：批量查名单

**布局：**
```
副标题：批量查名单
说明：上传含「学号」列的 Excel，系统会与当前生效名单比对，并支持下载命中结果。

[文件上传] 选择 Excel 文件（.xlsx / .xls）
[开始比对] 按钮

（比对后若有命中）
警告文案：⚠️ 共命中 {N} 条失信/违规记录，请核实。（当前第 x/y 页）
[表格] 列：姓名、学号、专业、原因、处分日期（当前页数据）

[分页区域]
  左：[上一页] 按钮（第一页时禁用）
  中：第 x 页 / 共 y 页，本页 z 条
  右：[下一页] 按钮（最后一页时禁用）

[下载比对结果报告 (Excel)] 按钮
```

**交互：**
- 未上传或未点击「开始比对」：不显示结果区
- 上传后点击「开始比对」：解析 Excel，与生效名单按学号比对；无学号时提示「Excel 中未找到有效的学号。」
- 无命中：成功提示「✅ 未查询到违规记录，名单内学生信用良好。」，不显示分页与下载
- 有命中：结果分页展示（每页 10 条），可翻页；下载按钮导出全部命中记录为 Excel

---

### 4.3 Tab2：名单管理

名单管理为长页，自上而下分为多个区块，用分割线与副标题区分。

#### 4.3.1 一、录入与导入 — 批量导入

```
说明：**一、录入与导入** — 从 Excel 批量写入或更新名单。
副标题：批量导入

[文件上传] 上传 Excel (.xlsx / .xls)

（上传并解析成功后）
说明：以下为解析结果前 10 行预览，确认无误后点击「开始导入」。
[表格] 解析结果前 10 行
[开始导入] 按钮
```

- 导入成功：成功提示 + 气球动效，并刷新页面
- 导入失败：错误提示（含已成功条数等信息）
- 学号为空的行会跳过并在结果中说明

#### 4.3.2 手动新增

```
分割线
副标题：手动新增
说明：单条录入，学号不可与已有记录重复。

[表单]
  姓名 [单行]
  学号 [单行]
  专业 [单行]
  原因 [多行]
  处分日期 [日期选择]
  [添加] 按钮
```

- 校验：姓名、学号必填；学号不可与已有重复
- 成功：「已添加。」并刷新

#### 4.3.3 二、查看与维护（生效名单）— 生效名单与删除

```
分割线
说明：**二、查看与维护（生效名单）** — 当前生效的失信记录，可筛选、导出、删除、编辑。
副标题：生效名单与删除

（无数据）暂无生效记录。
（有数据）
  说明：可按姓名、学号、专业筛选（留空表示不限制）。
  [筛选行] 三列：姓名筛选 | 学号筛选 | 专业筛选（文本输入，placeholder：留空不限制）

  [表格] 列：序号、姓名、学号、专业、原因（约50字）、处分日期（分页，每页 20 条）
  说明：当前第 x/y 页，本页 z 条（共 n 条）。
  [上一页] [下一页]

  [导出当前筛选的生效名单 (Excel)] 按钮

  输入要删除的学号 [单行，placeholder：学号]
  [软删除（设为已撤销）] 按钮
```

- 软删除：按学号将一条生效记录设为已撤销；未找到或学号无效时有对应错误提示

#### 4.3.4 初始化名单

```
分割线
副标题：初始化名单
说明：将所有生效记录设为已撤销，清空生效名单。请谨慎操作。

（未确认时）[初始化名单] 按钮
（点击后进入二次确认）
  警告：确定要初始化名单吗？此操作将把所有生效记录设为已撤销（软删除），生效名单将为空。
  [确认初始化] [取消]
```

#### 4.3.5 编辑现有人员

```
分割线
副标题：编辑现有人员
输入要编辑的学号 [单行]
[编辑] 按钮
```

- 点击编辑且找到记录后，进入「编辑表单」区块（见下）

**编辑表单（独立区块，按学号进入后显示）：**
```
说明：正在编辑记录 ID：{id}（学号不可修改）
[表单]
  姓名 [单行，预填]
  学号（不可修改）[只读/禁用]
  专业 [单行，预填]
  原因 [多行，预填]
  处分日期 [日期，预填]
  [保存修改] [取消]
```

- 保存后关闭编辑区块并刷新列表；取消仅关闭编辑区块并刷新

#### 4.3.6 三、已撤销名单 — 已撤销名单与恢复

```
分割线
说明：**三、已撤销名单** — 已撤销的记录，可筛选、导出、恢复为生效。
副标题：已撤销名单与恢复

（结构同「生效名单」：筛选三列 → 表格分页 → 导出 Excel → 恢复入口）

输入要恢复的学号 [单行]
[恢复为生效] 按钮
```

#### 4.3.7 四、查询与导出 — 批量查询

```
分割线
说明：**四、查询与导出** — 用 Excel 学号列与当前生效名单比对，并下载结果。
副标题：批量查询
说明：上传含「学号」列的 Excel，与生效名单比对；可下载比对结果报告。

[文件上传] 选择 Excel 文件
[开始比对] 按钮

（有命中时）
⚠️ 共命中 n 条失信/违规记录。
[表格] 姓名、学号、专业、原因、处分日期
[下载比对结果报告 (Excel)] 按钮
（无命中时）✅ 未查询到违规记录，名单内学生信用良好。
```

### 4.5 Tab4：系统维护

**布局：**
```
说明：查看操作日志、下载当前数据库备份，或从备份文件恢复（恢复前会自动备份当前库）。

副标题：审计日志
说明：可按操作人、操作类型、日期单独或组合筛选，留空或选「全部」表示不限制。

[筛选行] 三列
  操作人 [下拉：全部 + 用户列表]
  操作类型 [下拉：全部 + 类型列表]
  [ ] 按日期筛选 [勾选后显示] 选择日期 [日期]

（无数据）暂无符合条件的审计日志。
（有数据）
  [表格] ID、操作人、类型、对象、详情（约100字）、时间（最多 500 条）
  说明：共 n 条（最多展示 500 条）。
  [导出 CSV] [导出 Excel]

分割线
副标题：数据库备份下载
[下载当前数据库 (.db)] 按钮

分割线
副标题：⚡️ 危险操作：数据恢复 (Restore)
[文件上传] 上传备份文件以覆盖当前数据库（.db）

（未勾选）说明：请先勾选上方确认框后再执行恢复。
```

- 恢复前会自动执行一次当前库备份；恢复成功后提示并刷新页面

## 五、通用组件与交互模式汇总

| 模式 | 说明 |
|------|------|
| **表单** | 登录、手动新增、编辑、新增用户等均为表单；提交后校验，失败用错误提示，成功可刷新或局部更新 |
| **二次确认** | 初始化名单、数据恢复：先展示说明/警告，再通过独立按钮或勾选框确认后执行 |
| **分页** | 生效名单、已撤销名单、教师端批量比对结果：上一页/下一页 + 当前页/总页数 + 本页条数 |
| **筛选** | 生效/已撤销名单：姓名、学号、专业三列文本筛选；审计日志：操作人、类型、日期（可选） |
| **文件上传** | 批量导入（名单 Excel）、批量查名单/比对（学号 Excel）、数据恢复（.db）；上传后可有预览或解析结果再确认 |
| **下载按钮** | 导出生效/已撤销名单为 Excel、比对结果报告、审计日志 CSV/Excel、当前数据库 .db |
| **表格** | 多为只读数据表，占满容器宽度；列名见各区块说明 |
| **加载状态** | 查询、导入、重置等操作前可显示「正在查询...」「正在导入...」等 spinner |
| **反馈类型** | 成功（绿）、错误（红）、警告（黄）、信息（蓝）；重要操作成功可配合气球等轻动效 |

## 七、技术栈与实现备注（供设计还原参考）

- 当前实现为 **Streamlit**，单页应用，无多路由 URL；侧边栏固定左侧；主内容区为宽屏流式布局。
- 表格为 Streamlit 的 `st.dataframe`，支持横向滚动，无内置排序/列筛选（筛选通过上方独立控件完成）。
- 图表为 Plotly（饼图、柱状图），可替换为其他图表库，需保留「专业分布」「按处分年份分布」两个图表及数据含义。
- 水印为 HTML/CSS 注入，全屏固定、居中、旋转、低透明度，设计时可按「全屏装饰文字」处理。
